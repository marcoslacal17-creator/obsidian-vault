# 🧠 Vault de Marco

Sistema personal de conocimiento, proyectos y vida. Construido sobre Obsidian.

## Estructura

| Carpeta | Propósito |
|---|---|
| `00_Inbox/` | Captura rápida. Todo entra aquí y se procesa después. |
| `01_Daily/` | Notas diarias, semanales, mensuales (Periodic Notes). |
| `02_Projects/` | Proyectos activos con deadline y resultado claro. |
| `03_Areas/` | Áreas de responsabilidad continua (salud, finanzas, trabajo). |
| `04_Resources/` | Referencias, aprendizaje, temas. |
| `05_Archive/` | Inactivo, terminado, no se borra. |
| `06_Notes/` | Zettelkasten: Fleeting → Literature → Permanent. |
| `07_People/` | Personas, CRM ligero. |
| `08_Meetings/` | Notas de reuniones. |
| `09_Templates/` | Plantillas (Templater). |
| `10_Attachments/` | Imágenes y archivos. |
| `_Meta/` | Dashboards, MOCs, índices. |

## Filosofía
- **Inbox → procesar → carpeta correcta.** Cero notas huérfanas más de 7 días.
- **Una idea por nota** en `06_Notes/Permanent`.
- **Linkea todo.** El valor está en las conexiones, no en las notas sueltas.
- **Revisión semanal obligatoria** (Weekly review). Lo no revisado muere.

## Convenciones
- Frontmatter `type:` en TODA nota (daily, project, meeting, person, etc.)
- Tags en minúscula, sin acentos, con guiones (`#deep-work`)
- Nombres de fichero: `YYYY-MM-DD` para fechas, `Title Case` para conceptos
